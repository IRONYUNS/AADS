publish

Publish data using an MQTT connection

Syntax

publish(mqtt,topic,message)

publish(__,Name,Value)

Description

publish(mqtt,topic,message) publishes a message to topic in MQTT connection.

publish(__,Name,Value) uses additional options specified by one or more Name, Value pair arguments.

Examples

Publish Message on MQTT broker

%Create mqtt connection.
myMQTT=mqtt('tcp://broker.hivemq.com');

%Create topic and message.
topic='myDevices';
message="Door1 is open";

%Publish a message to a topic.
publish(myMQTT,topic,message);

Publish with QoS 2

publish(myMQTT,topic,message,'QoS',2);

Input Arguments

mqtt - Input MQTT connection, specified as a the mqtt object used to publish

topic - MQTT topic to use for publication specified as a string or character array

message - message to publish to a topic using MQTT specified as a string

Output Arguments

none

Name-Value Pair Arguments

Specify optional comma-separated pairs of Name,Value arguments. Name is the argument name and Value is the corresponding value. Name must appear inside single quotes (' '). You can specify several name and value pair arguments in any order as Name1,Value1,...,NameN,ValueN .

Use Name,Value arguments to specify parameter initial values and additional properties of publish function during model creation.

'QoS'

Quality of Service (QoS) parameter, specified as a comma-separated pair consisting of 'QoS' and an integer value. The three levels indicate that a message will be delivered:
• (0) at most once,
• (1) at least once,
• (2) exactly once.

Example: publish(mqtt,topic,message,'QoS',1);

'Retain'

Boolean or numeric equivalent indicating that the broker should retain the message for any new subscriber to the topic, specified as a comma-separated pair consisting of 'Retain' and a boolean value.

Example: publish(myMQTT,'myTopic',message,'Retain',true);

'Timeout'

Waiting time in seconds to complete a publish operation, specified as a comma-separated pair consisting of 'Timeout' and an integer value.

Example: publish(myMQTT,myTopic,'Timeout',10);
